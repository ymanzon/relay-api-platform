Relay es un API gateway autoalojado que hace proxy, mock, seguridad y monitoreo de cualquier endpoint HTTP — con un dashboard completo, sin archivos de configuración y sin dependencia de servicios externos.

Características

Todo lo que una API necesita,
nada de lo que no necesita.

Construido para desarrolladores que quieren control sin complejidad.

🔀
Motor de Proxy Dinámico
Enruta cualquier método HTTP a cualquier URL backend. Inyecta headers, resuelve secrets, configura timeouts. Sin archivos de configuración — todo vive en MongoDB.
🎭
Servidor de Mocks Instantáneo
Devuelve JSON personalizado, códigos de estado y delays artificiales sin necesitar un backend. Cambia a proxy real agregando una URL de destino.
🛡️
Seguridad por Endpoint
Rate limiting, allowlist/blocklist de IPs, device fingerprinting, firma HMAC, detección de amenazas (SQLi, XSS, path traversal) — todo activable por endpoint.
🔐
Vault de Secrets Cifrado
Guarda API keys, tokens y contraseñas cifrados con AES-256-GCM. Referéncialos como {{NOMBRE_SECRET}} en URLs y headers.
⚡
Scripts Pre/Post Evento
Ejecuta JavaScript antes (PRE) o después (POST) de cada llamada proxy. Inyecta headers, transforma peticiones, llama webhooks, registra en sistemas externos.
🌿
Environments y Variables
Define conjuntos de variables (dev, staging, prod) y cambia con un clic. Usa [[BASE_URL]] en tus endpoints — sin editar nada al cambiar.
📥
Importación desde OpenAPI
Crea endpoints en masa desde specs Swagger 2.x u OpenAPI 3.x. Pega JSON, sube un archivo o ingresa una URL. Params y mocks auto-poblados desde el spec.
📤
Exportación Multi-Formato
Exporta cualquier colección como Postman v2.1, Insomnia v4, OpenAPI 3.0 YAML o un script de carga k6. Comparte con tu equipo en segundos.
📊
Dashboard de Analíticas
Timeline de peticiones, latencia p95/p99, tasa de errores, endpoints más activos, drill-down por ejecución. 8 KPIs en tiempo real con indicadores de delta.
Cómo Funciona

El pipeline de proxy de 10 pasos.

Cada petición pasa por esta secuencia — cada paso puede bloquear, transformar o enriquecer la petición antes de que llegue a tu backend.

1
Búsqueda de Endpoint
Empareja la ruta virtual y el método HTTP contra los endpoints activos. Devuelve 404 si no hay coincidencia.
core
2
Validación de API Key
Si el endpoint requiere API key, verifica el header x-api-key o query param. Devuelve 401 si falta o es inválida.
seguridad
2.5
Pipeline de Seguridad
CORS → Blocklist/Allowlist de IP → Bloqueo por fingerprint → Límite de tamaño de body → Firma HMAC → Detección de amenazas → Rate limiting. Cada check puede devolver 400/401/403/413/429.
seguridad
3
Lectura de Caché
Busca en caché en memoria con clave derivada del ID de endpoint + query ordenada + body. En HIT, omite todos los pasos siguientes y devuelve la respuesta cacheada.
caché
3.5
Variables de Entorno
Resuelve los placeholders [[NOMBRE_VAR]] en la URL destino y headers usando el mapa de variables del environment activo.
entorno
4
Resolución de Secrets
Descifra e inyecta los valores de {{NOMBRE_SECRET}} en la URL destino y headers. Los valores nunca aparecen en logs ni en la base de datos.
cripto
5
Validación de Parámetros
Verifica que todos los params de query y body declarados como requeridos estén presentes. Los errores se registran en el log de ejecución.
core
6
Eventos PRE
Ejecuta los scripts o webhooks PRE adjuntos. Los scripts reciben ctx con la petición, el endpoint, el proxy de secrets y un mapa de extras mutable.
evento
7
Proxy o Mock
Si hay URL destino: reenvía la petición vía Axios y devuelve la respuesta real. Si no: sirve el mock configurado (con delay artificial opcional).
core
8–9
Eventos POST + Escritura de Caché + Log de Ejecución
Ejecuta scripts POST, escribe la respuesta en caché si aplica, y persiste el registro de ejecución completo en MongoDB para analíticas y auditoría.
evento
Ejemplos de Uso

Escenarios reales, resueltos.

Desde ocultar API keys hasta despliegues multi-entorno — Relay lo maneja.

01
Ocultar una API key de terceros
Tu frontend necesita datos de clima pero no puedes exponer tu API key en las peticiones del navegador.
# Guarda tu clave como Secret cifrado Secrets → Nuevo Nombre: WEATHER_KEY Valor: tu_api_key # URL Destino del endpoint https://api.weather.com/data? appid={{WEATHER_KEY}} # Llama desde el navegador — key nunca expuesta curl http://localhost:3000/api/clima
✓ Clave resuelta en el servidor · Invisible para el cliente
02
Mock API para desarrollo frontend
Tu equipo frontend está bloqueado esperando al equipo backend. Necesitas respuestas realistas ahora.
# Endpoint sin URL Destino = modo Mock Método: GET Ruta: usuarios/:id Body: { "id": "usr_1", "nombre": "María" } Status: 200 Delay: 150ms # Devuelve el mock al instante curl http://localhost:3000/api/usuarios/any
✓ Agrega URL Destino después para cambiar al backend real
03
Rate limiting para endpoint público
Un endpoint está abierto al público y necesitas prevenir abuso sin herramientas externas.
# Pestaña Security → Rate Limit Estrategia: Por dirección IP Peticiones Máx: 30 Ventana: 60 segundos Bloqueo: 300 segundos # Respuesta al exceder el límite HTTP 429 Retry-After: 300 X-RateLimit-Limit: 30 X-RateLimit-Remaining: 0
✓ Cada bloqueo queda en el log de auditoría con IP + fingerprint
04
Despliegue multi-entorno
Misma estructura de API en dev, staging y producción. Cambiar de entorno no debe requerir editar cada endpoint.
# Crea environments con la variable BASE_URL Dev: BASE_URL = http://localhost:8080 Stg: BASE_URL = https://staging.api.co Prod: BASE_URL = https://api.empresa.com # URL Destino del endpoint [[BASE_URL]]/v2/pedidos # Cambiar desde el selector del nav Sin editar ningún endpoint
✓ Combínalos: [[BASE_URL]]/v2?key={{TOKEN_SECRET}}
05
Importar desde Swagger / OpenAPI
Tienes un spec existente y quieres crear todos los endpoints en Relay sin hacer clic en formularios.
# Página Environments → Importar Spec # Pega JSON, sube archivo, o URL: https://petstore.swagger.io/v2/swagger.json # Clic en Preview, luego en Importar Resultado: ✓ 18 creados 2 omitidos (duplicados) Agrupados por tag → Colecciones Mocks del spec auto-poblados
✓ Compatible con OpenAPI 3.x y Swagger 2.x (JSON + YAML)
06
Verificar peticiones firmadas (HMAC)
Un microservicio interno llama tu endpoint y necesitas verificar que cada petición no fue manipulada.
# Security → HMAC Signing Algoritmo: HMAC-SHA256 Header: X-Signature Secret: (clic en Generar) # El cliente firma la petición: payload = METODO + \n + RUTA + \n + JSON(query) + \n + JSON(body) firma = HMAC('sha256', secret, payload) headers['X-Signature'] = firma
✓ Firma inválida → 401 HMAC_INVALID · Comparación resistente a timing attacks
Instalación

En funcionamiento en menos de 2 minutos.

No requiere Docker para desarrollo local. El tier gratuito de MongoDB Atlas funciona perfectamente.

1
Requisitos previos
# Node.js 18+ y npm
node --version   # debe ser v18 o superior
npm --version

# MongoDB 6+ local — O usa MongoDB Atlas (tier gratuito, sin tarjeta)
# Atlas: https://www.mongodb.com/atlas
2
Clonar e instalar dependencias
git clone https://github.com/your-username/relay-api-platform.git
cd relay-api-platform
npm install
3
Crear el archivo .env
cp .env.example .env
# Edita .env con tus valores:

MONGODB_URI=mongodb://localhost:27017/relay
PORT=3000
SECRET_ENCRYPTION_KEY=$(node -e "console.log(require('crypto').randomBytes(32).toString('hex'))")

# Para Atlas, reemplaza MONGODB_URI:
MONGODB_URI=mongodb+srv://usuario:pass@cluster.mongodb.net/relay
4
Iniciar el servidor
# Desarrollo (hot-reload)
npm run start:dev

# Producción
npm run build
npm run start:prod

# Abrir el dashboard
http://localhost:3000
