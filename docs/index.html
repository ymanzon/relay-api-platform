Features

Everything an API needs,
nothing it doesn't.

Built for developers who want control without complexity.

🔀
Dynamic Proxy Engine
Route any HTTP method to any backend URL. Inject headers, resolve secrets, set timeouts. Zero config files — everything lives in MongoDB.
🎭
Instant Mock Server
Return custom JSON, status codes, and artificial delays without a backend. Switch from mock to proxy by adding a Destination URL.
🛡️
Security Per Endpoint
Rate limiting, IP allowlists/blocklists, device fingerprinting, HMAC signing, threat detection (SQLi, XSS, path traversal) — all toggleable per endpoint.
🔐
Encrypted Secrets Vault
Store API keys, tokens, and passwords encrypted with AES-256-GCM. Reference them as {{SECRET_NAME}} in URLs and headers.
⚡
Pre/Post Event Scripts
Run JavaScript before (PRE) or after (POST) every proxy call. Inject headers, transform requests, call webhooks, log to external systems.
🌿
Environments & Variables
Define variable sets (dev, staging, prod) and switch with one click. Use [[BASE_URL]] in endpoint URLs — no edits needed when switching.
📥
OpenAPI Import
Bulk-create endpoints from Swagger 2.x or OpenAPI 3.x specs. Paste JSON, upload a file, or fetch from a URL. Parameters and mocks auto-populated.
📤
Multi-Format Export
Export any collection as Postman v2.1, Insomnia v4, OpenAPI 3.0 YAML, or a k6 load test script. Share with your team in seconds.
📊
Analytics Dashboard
Request timeline, p95/p99 latency, error rates, top endpoints, execution drill-down. 8 live KPIs with delta indicators vs. previous period.
How It Works

The 10-step proxy pipeline.

Every request passes through this sequence — each step can block, transform, or enrich the request before it reaches your backend.

1
Endpoint Lookup
Match the virtual path and HTTP method against enabled endpoints. Returns 404 if no match.
core
2
API Key Validation
If the endpoint requires an API key, verify the x-api-key header or query param. Returns 401 if missing or invalid.
security
2.5
Security Pipeline
CORS check → IP blocklist/allowlist → Device fingerprint block → Body size limit → HMAC signature → Threat detection → Rate limiting. Each check can return 400/401/403/413/429.
security
3
Cache Read
Check in-memory cache with key derived from endpoint ID + sorted query + body. On HIT, skip all remaining steps and return cached response.
cache
3.5
Environment Variables
Resolve [[VAR_NAME]] placeholders in the destination URL and headers using the active environment's variable map.
env
4
Secrets Resolution
Decrypt and inject {{SECRET_NAME}} values into the destination URL and headers. Values never appear in logs or the database.
crypto
5
Parameter Validation
Check that all declared required query params and body fields are present. Errors are recorded in the execution log but do not block the request.
core
6
PRE Events
Execute attached PRE event scripts or webhooks. Scripts receive ctx with the request, endpoint, secrets proxy, and mutable extras map.
event
7
Proxy or Mock
If a Destination URL is set: forward the request via Axios and return the real response. If not: serve the configured mock response (with optional artificial delay).
core
8
POST Events + Cache Write + Execution Log
Execute POST event scripts, write the response to cache if applicable, and persist the full execution record to MongoDB for analytics and audit.
event
Usage Examples

Real scenarios, solved.

From hiding API keys to multi-environment deployments — Relay handles it.

01
Hide a third-party API key
Your frontend needs weather data but you can't expose your API key in browser requests.
# Store your key as an encrypted secret Secrets → New Name: WEATHER_KEY Value: your_api_key # Set endpoint Destination URL https://api.weather.com/data? appid={{WEATHER_KEY}} # Call from browser — key never exposed curl http://localhost:3000/api/weather
✓ Key resolved server-side · Never visible to clients
02
Mock API for frontend dev
Your frontend team is blocked waiting for the backend team. You need realistic responses now.
# Create endpoint — no Destination URL Method: GET Path: users/:id Mock body: { "id": "usr_1", "name": "Jane" } Status: 200 Delay: 150ms # Returns mock instantly curl http://localhost:3000/api/users/any-id
✓ Add Destination URL later to switch to real backend
03
Rate limit a public endpoint
An endpoint is open to the public and you need to prevent spam and DDoS without external tooling.
# Security tab → Rate Limit Strategy: Per IP address Max Requests: 30 Window: 60 seconds Block Duration: 300 seconds # Response after limit exceeded HTTP 429 Retry-After: 300 X-RateLimit-Limit: 30 X-RateLimit-Remaining: 0
✓ Every blocked attempt logged with IP + fingerprint
04
Multi-environment deployment
Same API structure across dev, staging, and production. Switching targets shouldn't require editing every endpoint.
# Create environments with BASE_URL variable Dev: BASE_URL = http://localhost:8080 Stg: BASE_URL = https://staging.api.co Prod: BASE_URL = https://api.company.com # Endpoint Destination URL [[BASE_URL]]/v2/orders # Switch via nav bar dropdown No endpoint edits needed
✓ Combine with secrets: [[BASE_URL]]/v2?key={{TOKEN}}
05
Import from Swagger / OpenAPI
You have an existing API spec and want all 50 endpoints created in Relay without clicking through forms.
# Environments page → Import Spec # Paste JSON, upload file, or URL: https://petstore.swagger.io/v2/swagger.json # Click Preview, then Import Result: ✓ 18 created Skipped: 2 (duplicates) Grouped by tags → Collections Mock responses from spec examples
✓ Supports OpenAPI 3.x and Swagger 2.x (JSON + YAML)
06
Verify signed requests (HMAC)
An internal service calls your endpoint and you need to verify every request hasn't been tampered with.
# Security → HMAC Signing tab Algorithm: HMAC-SHA256 Header: X-Signature Secret: (click Generate) # Client signs the request: payload = METHOD + \n + PATH + \n + JSON(query) + \n + JSON(body) sig = HMAC('sha256', secret, payload) headers['X-Signature'] = sig
✓ Invalid signature → 401 HMAC_INVALID · Timing-safe comparison
Installation

Up in under 2 minutes.

No Docker required for local development. MongoDB Atlas free tier works perfectly.

1
Prerequisites
# Node.js 18+ and npm
node --version   # must be v18 or higher
npm --version

# MongoDB 6+ (local) — OR use MongoDB Atlas free tier
# Atlas: https://www.mongodb.com/atlas (free, no credit card)
2
Clone and install dependencies
git clone https://github.com/your-username/relay-api-platform.git
cd relay-api-platform
npm install
3
Create your .env file
cp .env.example .env
# Edit .env with your values:

MONGODB_URI=mongodb://localhost:27017/relay
PORT=3000
SECRET_ENCRYPTION_KEY=$(node -e "console.log(require('crypto').randomBytes(32).toString('hex'))")

# For Atlas, replace MONGODB_URI with:
MONGODB_URI=mongodb+srv://user:pass@cluster.mongodb.net/relay
4
Start the server
# Development (hot-reload on file changes)
npm run start:dev

# Production
npm run build
npm run start:prod

# Open the dashboard
http://localhost:3000
