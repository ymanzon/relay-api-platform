/environments — Environments

      
Define variable sets with [[VAR_NAME]] syntax · Switch active env from the nav bar

    

    

      
    

  


  
  

    How it works: Create an environment with variables (e.g. BASE_URL, API_VERSION).
    Use [[BASE_URL]] in your endpoint Destination URL or headers.
    The active environment's values are resolved at proxy time — before secrets ({{SECRET}}) are resolved.
    You can combine both syntaxes: [[BASE_URL]]/v[[API_VERSION]]/users?key={{MY_SECRET}}

            Response headers: X-RateLimit-Limit, X-RateLimit-Remaining, Retry-After — standard RFC 6585 format.
            Block duration = 0 means reject immediately without blocking future requests.
          

        


        
        

          

            

              
check_circle Allowlist (only these IPs may pass)

              
            

            

              
block Blocklist (always blocked)

              
            

          

          

            Supports exact IPs, CIDR notation (e.g. 192.168.0.0/16) and prefix wildcards (e.g. 10.0.).
            Blocklist is checked first, then allowlist. Empty allowlist = allow all.
            Real IP resolved from X-Forwarded-For → X-Real-IP → socket.
          

        


        
        

          
          

            Fingerprint = SHA-256 of IP + User-Agent + Accept-Language + Accept-Encoding + Sec-CH-UA headers.
            A 16-char identifier unique to each client combination. Useful as a rate-limit strategy or for manual blocking.
          

          
block Blocked Fingerprints

          
          

            Fingerprint appears in security event logs and execution meta. Use it to identify repeat offenders.
